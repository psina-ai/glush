## 1. Модель данных

### 1.1 Шкалы

Три шкалы, все в диапазоне **[−100, +100]**. Стартовое значение: **0**.

| ID (enum) | Русское название | Минус (−100) | Плюс (+100) |
|---|---|---|---|
| `Zhiteyskaya` | Житейская | Эхо войны | Фатализм |
| `Povedencheskaya` | Поведенческая | Приказ командира | Своеволие |
| `Poetichnaya` | Поэтичная | Трезвость | Иллюзия |

### 1.2 C#-модель (эскиз, не финальный код)

```csharp
public enum PersonalityAxis {
    Zhiteyskaya,       // главная — определяет путь и финал
    Povedencheskaya,   // разблокировщик противоречий + модификатор тона
    Poetichnaya        // косметика + гейт побочного контента
}

// ScriptableObject-настройки баланса (сдвиги, пороги)
[CreateAssetMenu(...)]
public class PersonalityBalanceSettings : ScriptableObject { ... }

// ScriptableObject-состояние (текущие значения шкал)
[CreateAssetMenu(...)]
public class PersonalityState : ScriptableObject {
    public int zhiteyskaya;
    public int povedencheskaya;
    public int poetichnaya;
}

// Публичный API — единственная точка входа
public static class PersonalitySystem {
    public static int  GetValue(PersonalityAxis axis);
    public static void ApplyShift(PersonalityAxis axis, int delta);
    public static bool IsChoiceAvailable(ChoiceRequirement req);
    public static ToneVariant GetCurrentTone(); // на основе Povedencheskaya
    public static void ResetToStart();
    public static event Action<PersonalityAxis, int, int> OnValueChanged; // (axis, old, new)
}
```

## 2. Правила доступности вариантов ответа

Правила применяются в момент, когда движок Ink предлагает набор `choices`.
Каждый Ink-choice может иметь тег `#requires:...` — фильтр.

### 2.1 Базовое правило (по шкале №1 Житейская)

Пусть `Z = |PersonalityState.Zhiteyskaya|`.

| Уклон | Доступно |
|---|---|
| **Z ≤ 20** («у распутья») | Нейтральный + оба окрашенных (влево и вправо) |
| **20 < Z ≤ 60** («умеренный уклон») | Нейтральный + вариант «в тон уклону» |
| **Z > 60** («сильный уклон») | **Только вариант «в тон уклону»** (нейтральный убирается) |

Направление уклона определяется по знаку `PersonalityState.Zhiteyskaya`:
- Знак `<0` → уклон в «Эхо войны»
- Знак `>0` → уклон в «Фатализм»

### 2.2 Модификатор от шкалы №2 (разблокировка противоречий)

**Триггер:** мы находимся в диапазоне «умеренного уклона» (`20 < Z ≤ 60`) → противоположный вариант формально скрыт.

**Условие разблокировки:**
- Шкала №2 (Поведенческая) сильно отклонена **в направлении, противоположном** доминирующему направлению шкалы №1.
- «Сильно» = `|Povedencheskaya| > 40`.
- «Противоположно» = знак Поведенческой обратен знаку Житейской.

**При выполнении** — скрытый противоположный вариант становится **доступен** и помечается флагом `IsContradictoryChoice = true` (чтобы UI мог его особо оформить).

**Пример:**
Житейская = +30 (мягкий Фатализм) → нормально доступен нейтральный + вариант «в Фатализм».
Поведенческая = −50 (сильный Приказ) → знаки противоположны, |Поведенческая| > 40 → **разблокируется** «противоречивый» вариант в сторону Эха («взять себя в руки, отставить меланхолию»).

### 2.3 Правила для шкалы №3 (Поэтичная)

**Обычные диалоги:** правила из §2.1–2.2 применяются только к шкале №1. Шкала №3 не влияет на доступность.

**«Глубокие» диалоги** (помечены в Ink тегом `#deep_dialogue`):
- Работают те же правила §2.1, но применяются к **шкале №3**, а не к №1.
- Модификатор от Поведенческой (§2.2) в глубоких диалогах не применяется (Поведенческая — не про мышление).

## 3. Тон реплик (окраска)

Отдельный механизм, не связанный с доступностью.

### 3.1 От шкалы №2 (Поведенческая)

Активен при `|Povedencheskaya| > 40`.

| Значение | Тон |
|---|---|
| `Povedencheskaya < −40` | **Сухой / Дисциплинированный** — минимум слов, никаких эмоций |
| `−40 ≤ Povedencheskaya ≤ 40` | **Нейтральный тон** |
| `Povedencheskaya > +40` | **Грубый / Дерзкий** — сарказм, ирония, вызов |

### 3.2 От шкалы №3 (Поэтичная)

Работает всегда, накладывается поверх тона от Поведенческой.

| Значение | Тон |
|---|---|
| `Poetichnaya < −40` | **Приземлённый** — сухие факты, депрессивный сарказм |
| `−40 ≤ Poetichnaya ≤ 40` | **Обычный** |
| `Poetichnaya > +40` | **Поэтичный** — метафоры, лирика, «странности речи» |

### 3.3 Реализация в Ink

Каждая реплика в Ink пишется в базовой форме, но может иметь **альтернативные варианты**:

```ink
* [Молчать] -> silence
* [Ответить] 
    { tone_pov == "harsh": "Хер с ним, куда торопиться-то." 
    | tone_pov == "dry":   "Всё равно. Спешить некуда." 
    | else:                "Да ладно, спешить некуда." }
    -> continue
```

Переменные `tone_pov`, `tone_poet` устанавливаются движком **автоматически** перед каждым узлом на основе шкал (см. §3.1, §3.2).

## 4. Ink-теги (контракт между дизайнером и кодом)

Игра ищет эти теги на репликах и выборах Ink:

### 4.1 Теги на выборах (Choice)

| Тег | Значение |
|---|---|
| `#req:AXIS:>=VAL` или `#req:AXIS:<=VAL` | Формальное требование на шкалу для показа выбора |
| `#tone:pole` | Обозначает, к какому «полюсу» относится этот выбор. `pole` ∈ `{echo, fatalism, order, willful, sober, illusion, neutral}` — используется движком для правил §2 |
| `#shift:AXIS:±N` | Сдвинуть шкалу при выборе этого варианта |
| `#deep` | Помечает выбор как относящийся к «глубокому диалогу» (§2.3) |
| `#contradictory` | Маркер, что выбор — это «противоречивая» разблокировка (для UI) |

`AXIS` ∈ `{ZHIT, POV, POET}` (короткие имена для удобства записи).

### 4.2 Теги на узлах (Knot / Stitch)

| Тег | Значение |
|---|---|
| `#deep_dialogue` | Весь узел — «глубокий» (см. §2.3) |
| `#npc:NAME` | Кто говорит |
| `#emote:MOOD` | Эмоция (для будущей системы портретов) |

### 4.3 Пример Ink-фрагмента

```ink
=== pyaterochka_first_visit ===
#npc:Кассирша
Кассирша медленно бьёт штрих-код сигарет.
Пиип. Пауза. Пиип.

* [(Нейтрально) Ждать молча] #tone:neutral #shift:ZHIT:+1
    -> wait
* [Раздражённо] Ну быстрее нельзя? #tone:echo #shift:ZHIT:-3
    -> annoyed
* [Философски] А чего спешить. Мы все — в очереди. #tone:fatalism #shift:ZHIT:+3 #shift:POET:+2
    -> philosophical
* [(Скрытый глубокий) Штрих-код как молитва...] #deep #tone:illusion #shift:POET:+5
    -> deep_reading
```

Первые три варианта — обычные, фильтруются по правилам §2.
Четвёртый (`#deep`) — не показывается вообще, если Poetichnaya < 30.

## 5. Экономика (значения по умолчанию)

Все хранятся в ScriptableObject `PersonalityBalanceSettings`.

```
Малый выбор в тон:                ±2
Средний моральный выбор:          ±6
Ключевое сюжетное действие:       ±12
Убийство мирного:                 −20 (в Эхо)
Убийство дружелюбного монстра:    −15 (в Эхо)
Убийство агрессивного (нужное):     0
Убийство агрессивного (лишнее):   −2  (в Эхо)
Пощада агрессивного:              +3  (в Фатализм)
Ритуал (медленный дрейф):         +1  (в Фатализм)
```

**Клэмпинг:** после каждого сдвига значение шкалы приводится к диапазону [−100, +100].

**Round-trip тест (для проверки баланса):**
- 100 малых выборов × ±2 = ±200 → клэмп в ±100
- Норм. — за 5-8 часов игрок должен успевать «войти в характер», но не «перескочить» его случайно.

## 6. Псевдокод основной проверки

```csharp
// Вызывается, когда Ink отдаёт список Choice
bool IsAvailable(InkChoice choice) {
    var req = ParseTags(choice.tags); // req.axis, req.tonePole, req.formalRequirements
    
    // 1. Формальные req:AXIS:>=VAL — самый жёсткий фильтр
    if (!req.formalRequirements.All(r => Check(r))) return false;
    
    // 2. Глубокие диалоги идут по своей оси (Poetichnaya)
    var axis = req.isDeep ? PersonalityAxis.Poetichnaya : PersonalityAxis.Zhiteyskaya;
    int value = PersonalitySystem.GetValue(axis);
    int abs = Math.Abs(value);
    int sign = Math.Sign(value);
    var pole = req.tonePole;
    
    // 3. Нейтральный вариант
    if (pole == Pole.Neutral) {
        return abs <= 60; // при сильном уклоне нейтральный уходит
    }
    
    // 4. Вариант «в тон уклону»
    bool inLine = (sign > 0 && pole.IsPositive()) || (sign < 0 && pole.IsNegative());
    if (inLine) return true; // всегда доступен
    
    // 5. Вариант «против уклона»
    if (abs <= 20) return true; // распутье — всё видно
    if (abs > 60) return false; // жёсткий уклон — нет
    
    // 6. Умеренный уклон (20..60) — противоположный вариант нормально скрыт,
    //    НО может быть разблокирован Поведенческой (§2.2)
    if (!req.isDeep) {
        int pov = PersonalitySystem.GetValue(PersonalityAxis.Povedencheskaya);
        if (Math.Abs(pov) > 40 && Math.Sign(pov) != sign) {
            return true; // противоречивая разблокировка
        }
    }
    return false;
}
```
